CMainSystemBase¶

The CMainSystemBase added to the game object . The game object is not deleted even switched the scene

By adding such as resident manager through this script , download the related asset .

In addition , the pending switching , such as during AssetBundle download With this script to the scene switching .

When using it , create a CMainSystem class and to inherit it .

void Awake()¶

In Awake By calling the AddComponent and addManager, and you give me a read automatically to asset bundles with the necessary initialization and in the Start.

A typical description method is as follows .

Note

Start () should not be overwritten . It can not be successfully loading .

new void Awake() {
       base.Awake();

    if (m_instance != null) {
        Debug.LogError("already exist CMainSystem");
        return;
    }
    m_instance = this;

    // Add Component
    gameObject.AddComponent<CInput>();
    gameObject.AddComponent<CAssetBundleMgr>();
    gameObject.AddComponent<CSpriteFontMgr>();
    gameObject.AddComponent<CTextureResourceMgr>();
    gameObject.AddComponent<CWindowMgr>();
    gameObject.AddComponent<CBgmResourceMgr>();
    gameObject.AddComponent<CSeResourceMgr>();
    gameObject.AddComponent<CSoundEffectMgr>();

    // 初期化の必要なマネージャ.
    addManager(new CMessageDataSheetMgr(Utility.getSystemLocale()));
}
void addManager(IManager mgr)¶
Parameters IManager mgr:
 Objects must be initialized by loading the asset bundles

Note

In Awake, it is necessary to AddComponet.

virtual void initialize()¶

There is a need to override in the inheritance destination . After the asset bundles of load , load of ILoader and IManager completed , it is called.

virtual void OnChangeScene()¶

Override inheritance destination . Called when you switch the scene .

bool changeScene(string sScene, bool bForceChange = false)¶

It is switched to the specified scene .

It is equivalent to the Application.LoadLevel , but it deferents following points .

  1. Wait until the fade of 「fade object」 is finished .

  2. While asset bundles loading , if you are switching a scene ,pending it.

  3. The created windows close automatically.

To the scene immediately before switching ( just before the Application.LoadLevel is called ), OnChageScene is called .

bool isChangingScene¶

Determining whether or not in the scene load .

It is effective only when you are switching the scene by CMainSystemBase.chageScene.

bool isInitialized¶

It is determined whether it has been completed the initialization.

static int randi()¶

You get a random number of integer values.

static int randi(int iMax)¶

You get a random number in the specified range.

static int randi(int iMin, int iMax)¶

You get a random number in the specified range.

static float randf()¶

You get a random number of floating point.

ILoader interface¶

bool isLoading()¶
Return bool:

Loading is true,the end of load is false .

There is a need to put on the inherited script MonoBehaviour.

Objects are valid with ILoader intarface, it is necessary to wait until the value of ILoader.isLoading () is false.

After waiting the end , it start to load objects that you added by addManager function.

When the initialization of IManager ends , CMainSystemBase.initialize () is called. And CMainSystemBase.isInitialized turns true.

Note

In Awake, it is necessary to AddComponet.

IManager interface¶

Awake () within these should be added using the CMainSystemBase.addManager.

It is an interface required for the objects that must be initialized with the loaded asset bundle.

addManager(new CMessageDataSheetMgr(Utility.getSystemLocale()));
