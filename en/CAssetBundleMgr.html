CAssetBundleMgr¶

Call AddComponent in Awake of CMainSystem.

This is the manager to cache or load the asset bundle.

When you prepare your own manager , it is unnecessary .

First , download the version.unity3d, to get the version of each asset .

Discard the data cached locally when the versions are different , it re- downloaded .

To have the following characteristics .

  1. Asset bundles are version control , once the imported ones so that load from the cache (WWW.LoadFromCacheOrDownload)

  2. The loaded asset bundles are cached in memory.

  3. It releases the cache at the time of scene switching .

  4. An asset bundle resident flag is attached to hold it without releasing.

  5. Possible to switch the load path in the configuration file on the MS-Windows and the Mac ( local files without using the HTTP server can also be loaded)

  6. The name of the asset bundle limited to format that MulID .unity3d.

CAssetBundleMgr¶

const int MaxConcurrentLoadNum¶

The maximum number to be downloaded at the same time

It is the maximum number that at the same time calling a LoadFromCacheOrDownload.

The default value is to 1.

const long MaximumAvailableDiskSpace¶

It sets the maximum cache size to a local disk.

The default value is set to 512 * 1024 * 1024 (512Mbyte).

static string httpServerPath¶

It sets the HTTP address.

static string debugPath¶

It sets the debug path . It is effective only in MS-Windows / Mac environment . In favor of the set path to load the file .

Note

A file that was loaded via the debug path does not perform the cache to the local disk.

int lastError¶

It has the error number

int errorMessage¶

It has the error message

CAssetBundle[] lodings¶

This returns a list of asset bundles currently being downloaded .

This is also included in this list that are pending download therein.

int loadNum¶

It has a number of asset bundle currently being downloaded .

This is also included in this list that are pending download therein.

CAssetBundleMgr Instance¶

It has an instance of a manager

CAssetBundle reference(uint id)¶

It refers to the asset bundle. If you have not cache it , the manager download it .

When the asset bundle exist without already downloaded , it returns the CAssetBundle.

Whether CAssetBundle is being downloaded that have been returned , it is possible to determine by CAssetBundle.isLoaded.

When it returns true, it is enabled .

When null is returned , it is when the asset bundle not exist .

bool isExist(uint id)¶

It examines whether the asset bundle exist .

void startPreload()¶

When you call , to download all asset bundles that exist in order to cache on the local disk

void loadAssetVersion()¶

Asset version (version.unity3d) to download and again , and updates the cache of asset bundle.

void releaseAll()¶

It releases all asset bundle regardless of the resident / non-resident .

After the release , you must call the loadAssetVersion.

CAssetBundle¶

It can be cast in AssetBundle.

AssetBundle get()¶

Get the AssetBundle.

uint id¶

It has the ID.

bool isLoaded¶

true if it is already loaded .

bool isRemain¶

Check whether It is resident .

The path how to change a local file to load¶

It is only valid on Windows / Mac.

You make the following files in the same location as the Assets folder.

  • config.cf
  • config.local.cf

Order to be loaded is config.cf => config.local.cf.

It is overwritten those load later .

It is described as follows in config.cf

#
# config.cf
#
DEBUG_PATH = ../../../unified/AssetBuilder/assetbundles/Windows

In the Awake of CMainSystem, set as follows .

//==========================================================================
/*!Awake
* @brief    Unity Callback
*/
new void Awake() {
     base.Awake();
     if (m_instance != null) {
         Debug.LogError("already exist CMainSystem");
         return;
     }
     m_instance = this;
     #if !(UNITY_IPHONE || UNITY_ANDROID || UNITY_WEBPLAYER)
         CAssetBundleMgr.debugPath = CConfig.Instance.get ("DEBUG_PATH","../Debug/");
     #endif
     gameObject.AddComponent<CAssetBundleMgr>();
